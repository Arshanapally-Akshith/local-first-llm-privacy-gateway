## THE PROJECT

A local-first egress proxy that sits between an application and a cloud LLM provider. Outbound requests are inspected and sanitized: sensitive entities are replaced with **realistic, format-preserving surrogates** before leaving the machine. The streaming response is **rehydrated** locally so the caller sees real values back. The provider never sees the real data.

**The one-line pitch:**
```python
client = OpenAI(base_url="http://localhost:8080/v1", api_key="...")
```
One line changes. Prompts stop leaking.

### What actually differentiates this project

Two things, and only two:

1. **A fairly-baselined Indian-PII benchmark.** Presidio configured *with* custom recognizers, not stock. Ablation arms so the delta is attributable.
2. **An adversarial bypass suite that reports the bypasses that still work.**

Everything else — the gateway pattern, FPE surrogates, reversible pseudonymization — is a competent reimplementation of things Skyflow and Google Cloud DLP ship in production. That is fine. It is claimed as such. **Any language in this repo implying novelty for the gateway itself is a bug and you will remove it.**

**The evaluation is the project. The proxy is table stakes.** If we fall behind, we ship a worse detector — never a worse benchmark.

## PHASE 2 — Tier 1 Detection + FF1 Surrogates (REQUEST PATH ONLY)

- **Tier 1 detectors**, checksum-validated where the format allows:

  - Card (**Luhn**), email, Indian phone
- **FF1 (NIST SP 800-38G)** surrogates over the correct radix/domain per entity type. Keyed, stateless, invertible. **No map for Tier 1.**

## PHASE 3 — Session Map, Rehydration, Multi-Turn Integrity

The hard phase. Do not rush it.

- **Session-scoped in-memory map** for Tier-2-class entities (built now, populated in Phase 4). TTL, explicit eviction, **thread-safe under concurrent requests on the same session** — two in-flight requests assigning a surrogate to the same new name must not produce two mappings.
- **Collision handling at assignment time.** With a finite name list, birthday collisions are likely, not theoretical. Two real people must never map to one surrogate. Test with a forced-tiny name list to make collisions certain.
- **Rehydration through the sliding window.** Surrogates arriving split across chunks must be reassembled and replaced. Use the Phase-1 pathological chunker to prove it.
- **Ingress surrogate recognition.** Turn 3's request contains turn 1's assistant message, which already contains surrogates. Detect and pass through — **do not re-encrypt**. A surrogate-of-a-surrogate unwinds one layer and corrupts silently. This never fires in single-turn tests, so it needs an explicit multi-turn test.
- **Rehydration-fidelity harness.** Measure, don't fix: what fraction of surrogates come back in a form we can match? Build the taxonomy now — exact, partial (`Arjun` only), decorated (`**Arjun Reddy**`), case-shifted, abbreviated (`A. Reddy`), transliterated, reasoned-about ("the name starts with A"). Report per-category rates.
- **Matching is conservative by design.** Aggressive fuzzy matching turns the proxy into a **rehydration oracle** — an attacker who learns the surrogate distribution gets us to reinsert real PII into attacker-readable output. Document this tradeoff in DECISIONS.md. Conservative matching means visible misses; that is the correct choice and we will say so.

**Definition of Done**
- [ ] Concurrency test: 50 parallel requests on one session, no duplicate/lost mappings
- [ ] Collision test with a forced 3-name list: no two entities share a surrogate
- [ ] Split-surrogate rehydration passes across 1/2/3/N chunk splits
- [ ] Multi-turn test: 5-turn conversation, no double-encryption, no corruption
- [ ] Rehydration-fidelity harness runs and emits per-category numbers to an artifact
- [ ] Session TTL eviction tested; map is empty after expiry
- [ ] Oracle tradeoff written into DECISIONS.md
- [ ] Summary + state updated

**Gate:** A 5-turn conversation through the proxy, mock upstream forced to split every surrogate across 3 chunks and to echo names in decorated/partial forms. Round-trip stays correct where matchable; misses are *reported*, not silently swallowed.

## PHASE 5 — The Benchmark (FIRST-CLASS DELIVERABLE)

**This is the project.** Not a chapter of it.

- **Generation methodology — non-negotiable:** LLM generates **carrier sentences with slots** (`"mera address {ADDRESS}, contact {PHONE}"`); entities are injected **programmatically**. Gold offsets are exact by construction. Do **not** ask an LLM for sentences *and* spans — you will get 10–20% misaligned offsets and spend a week hand-correcting a dataset instead of building.
- ~2–3k examples, ~10 entity types, English + Hinglish/Telugu-English code-switched, including real-world mess: `"PAN no. ABCDE1234F"`, spaced/dashed digits, transliterated names, OCR-ish noise.
- **Honest framing, in the dataset card:** carrier phrasings are LLM-diverse; **entity surface forms are ours**. Recall against our own generator is an **optimistic bound**. Say this in the README, not just the code.
- **Fair Presidio baseline.** Presidio ships no Aadhaar/PAN/IFSC/UPI recognizers. Beating stock Presidio is beating a tool at a task it does not attempt, and one question kills the claim. **Commit the custom recognizer config.**
- **Ablation arms — all four:**
  1. Presidio, stock
  2. Presidio + our custom recognizers
  3. Presidio + custom recognizers + GLiNER backend
  4. Ours (full cascade)

  If arm 3 ≈ arm 4, the honest finding is *"the cascade buys latency, not accuracy."* That is a **good** result. Report it. Do not tune until arm 4 wins.
- **Span-matching criterion fixed before measuring.** Exact-span vs token-level vs partial credit differ by 10+ points. Pick one, justify it in DECISIONS.md, apply it identically to all four arms.
- `make bench` regenerates every number in the README. Per-entity P/R/F1, per arm.
- Publish: dataset + card, **synthetic-only stated loudly**, reserved-range Aadhaar note.

- [ ] Dataset card written: synthetic-only, generation method, optimistic-bound caveat, reserved-range note

## PHASE 7 — Latency Harness

Three specific traps; instrument against all of them.

- **Measure under concurrency.** A single-threaded loop gives a fantasy p99. Python GIL + CPU inference at 4 concurrent requests is a different distribution entirely. Report the concurrency level with every number or don't report p99.
- **Cold start reported separately.** First GLiNER call is seconds. In p50 → the number is wrong. Silently dropped → we're hiding the worst UX moment.
- **TTFT is the only latency a human perceives**, and our buffered sliding window taxes it. Report TTFT separately from total.
- Per-tier p50/p95/p99 + tier-hit distribution. **Tier-hit rate is a property of our dataset**, not of the world — real traffic is mostly PII-free. State it as *measured-on-benchmark*.

**Definition of Done**
- [ ] Harness runs at 1 / 4 / 8 concurrency; level reported with every number
- [ ] Cold start excluded from steady-state and reported as its own line
- [ ] TTFT reported separately from total latency, with and without the window
- [ ] Per-tier p50/p95/p99 + tier-hit distribution, regenerated by `make bench`
- [ ] "measured-on-benchmark, not real traffic" caveat in the table caption
- [ ] Summary + state updated

**Gate:** The latency table states its concurrency level and its TTFT tax without me asking.

## THINGS THAT WILL GO WRONG — DESIGN FOR THEM NOW

- **The biggest risk is schedule shape, not tech.** Phases 1–4 are fun (SSE, chunk boundaries, FPE) and will eat the calendar. Then the benchmark and adversarial suite get compressed into the last week, and we ship a competent Skyflow reimplementation with no differentiation — which is a rejection. **If week 6 arrives and we're behind: ship a worse detector, never a worse benchmark.**
- **FF1 does not cover names.** FPE is a permutation over a fixed domain. Aadhaar is fixed; arbitrary Unicode names are not. Realistic name surrogates require a finite list and a one-way index map. This is why the session map exists and why "no vault" is only true of Tier 1. Do not let this surprise us in Phase 4.
- **Rehydration is exact-string matching against a model that does not preserve strings.** Expect misses. Measure them; don't fuzzy-match your way into an oracle.
- **Multi-turn corruption is invisible in single-turn tests.** Ingress surrogate recognition is not optional.
- **Benchmark labels are the classic time sink.** Slot-and-inject is the mitigation. Do not deviate.
- **Cold start will pollute p50** if the model isn't warmed and the first call isn't excluded deliberately.
- **Our own logs are a leak surface.** The Phase-0 formatter exists for this reason.
